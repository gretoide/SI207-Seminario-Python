Enunciado de la pregunta

Escribí un programa en C que se comporte como el comando tr de los sistemas operativos *nix:

El comando tr se utiliza para transformar strings, principalmente traduciendo los caracteres de una lista, que aparecen en el string de entrada, en los de otra lista; borrando las ocurrencias de una lista de caracteres en el string de entrada; o eliminando repeticiones de caracteres, entre otras posibilidades.

Por simplicidad, nos enfocaremos en las primeras dos variantes de uso del comando enumeradas:

  1. Traducción de caracteres de una lista hacia los de otra.
  2. Borrado de caracteres de una lista.

A continuación se detalla la forma de invocar cada opción y cómo debe comportarse el programa en cada caso. De aquí en más, denominaremos a nuestro programa tb.

En todos los casos, el programa leerá de la entrada estándar stdin el string ("la entrada") que deberá procesar, y deberá imprimir a la salida estándar stdout el resultado.

Variante 1: traducción (-t)

Esta variante hace que el programa cambie la entrada reemplazando los caracteres que encuentre de una lista "de origen" (la primera que aparezca en orden de los argumentos que el programa recibe) por su caracter correspondiente de una segunda lista "de destino". La correspondencia se hace por posición: el primer elemento de la lista de origen se reemplaza por el primer elemento de la lista de destino, el segundo de la lista de origen por el segundo de la lista de destino, y así sucesivamente. Esta acción se denota con el argumento -t y requiere que se pasen adicionalmente dos strings, uno para cada lista de caracteres (origen y destino).

Ejemplo:

$ echo 'generador de passwords' | tb -t "eaosr " "3405R."
g3n3R4d0R.d3.p455w0Rd5

Variante 2: borrado (-b)

En esta variante, el programa deberá borrar toda ocurrencia que encuentre en la entrada de algún caracter de la lista que se le pasa como argumento. Esta acción se denota con el argumento -b y requiere que se pase adicionalmente un string que es la lista de caracteres a borrar de la entrada.

Ejemplo:

$ echo "un string Bastante interesante con 1, 2, 3 y 4 numeros" | tb -b aBcsd3
un tring tnte interente on 1, 2,  y 4 numero


Notas

  1. Si no se especifica ninguna acción a realizar (opción -t o -b), el programa debe informar esto en la salida de error estándar stderr y terminar su ejecución con un valor de retorno 1.
  2. Si no se especifica alguno de los argumentos requeridos de la acción a realizar, se debe informar esto en la salida de error estándar stderr y terminar la ejecución del programa con un valor de retorno 2.
  3. (Únicamente para la variante de traducción) Si la longitud de la lista de caracteres de origen y destino difieren, el programa debe informar esto en la salida de error estándar stderr y terminar su ejecución con un valor de retorno 3.

Comentarios

Comentario:

Compila sin warning ni errores.

No se procesa las opciones como argumentos al programa.

En la carga de texto a procesar como los argumetos para la opción t se asume que van a terminar con '\n'.

Las variable l, src, dest se tratan como arreglos de chars no como string dado que no tienen el carcater final '\0'.

En los whiles se tiene una comparación del contenido de los arreglos con '\n' siendo que el mismo valor nunca es cargado al arreglo porque es el corte. Además, la iteración se hace mientras <= sin tener en cuanta que las variables contaron el ingreso del '\n'. Se debía usar el < para no acceder incorrectamente.

Se usa printf para la impresión de un caracater. Para estos casos es conveniente el uso de putchar.

No se verifica que src y dest tengan igual cantidad de caráteres.

No se entrega el párametro -b.

Cuando se no se ingresa t en el main el else obliga a que se ingrese un nuevo caracter en lugar de procesar el utilizando en la comparación con t.

No se hace uso de strerror ni se returna como final de programa los valores indicados en el enunciado para cada caso.
